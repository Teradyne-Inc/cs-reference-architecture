Auto-detect the ssn-setup pattern module

by iterating through pattern modules and looking for pattern vector tags: the first pattern module that contains contribution bit tags will be recognized as the pattern that needs patching.

string GetSsnSetupPatternModuleName(string[] moduleList, out IVectorTagIdsCollection contribTags) {
    contribTags = null;
    foreach (string moduleName in moduleList) {
        contribTags = TheHdw.Digital.Patterns().Modules[moduleName].Tags.GetTagIdsAll("disable_on_chip_compare_contribution");
        if (contribTags != null && contribTags.Count > 0) {
            return moduleName;
        }
    }
    return moduleList.FirstOrDefault();
}

Bug Fixes

fix bug in IGComponentManagerParser.cs. support in the new Oasis version 4.6.10801 support in the IG-Component manager Schema that look like: { "Publisher": "Teradyne", "Name": "Oasis", "Comparison": "Equal to", "Version": "4.6.10801", "Notes": "", "Verification": "Default", "Location": "" }, This pull request introduces significant improvements to the IGComponentManagerParser and its related infrastructure, focusing on enhancing version verification logic, JSON schema validation, and test coverage. The changes modernize and generalize version comparison, add robust structure validation for configuration files, and expand the supported set of Teradyne components.

Key changes include:

IGComponentManagerParser enhancements

  • Generalized version comparison logic:
    The VersionCompare method now supports a wider range of comparison operators (Equal to, Greater than, Greater than or equal to, Minimum, Between), handles version ranges, and cleans version strings more robustly. This enables more flexible and accurate validation of component versions.
  • Component-specific verification methods:
    Added verification support for new Teradyne components such as PortBridge and Visual Studio. Each now has a dedicated method to extract and compare their versions, using environment variables or system tools as appropriate.
  • Improved error handling and feedback:
    If required environment variables are missing or files are not found, explicit error messages are logged. This ensures that missing prerequisites are clearly reported.

Adding exceptions for PPMU pin type

IG-XL instruments does not support having ReadSamples or StrobeSamples for PPMU pin types. These methods will now throw an exception when these methods are called for PPMU.

Impact: What is the impact for the user? User will now see the method throw an exception if trying to use ReadMeasuredSamples or StrobeSamples for PPMU pin types.

Fix DCVS voltage range validation when meter mode is Voltage

Corrected the voltage range value handling/validation for DCVS when the meter mode is set to Voltage, preventing the UI/runtime error triggered by an invalid or mismatched range value.

Impact: What is the impact for the user? Users can set Meter Mode = Voltage on DCVS without encountering voltage range errors; voltage range values are consistently validated/mapped to supported ranges, improving reliability in configuration and execution.
